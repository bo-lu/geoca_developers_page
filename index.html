- Harmonized North American Profile of ISO 19115:2003 | A focus on metadata standards and content | # Architecture The diagram shows a high-level architecture of geo.ca, which includes JavaScript, React.js, YAML, JSON and other configurations files which defines the ‘infrastructure as code’. A content management system then uses this infrastructure and accepts templates and content as input and outputs static HTML and CSS files on geo.ca. Contributor metadata are converted to a GeoCore format based on GeoJSON which can then be displayed using the GeoView and are harvestable using an OGC CSW or OGC API – Record endpoint. Contributors are able to publish web services using ESRI ArcGIS Enterprise or MapServer. Finally, geo.ca has developed a suite of microservice applications which are accessible using modern REST APIs. # Component documentation
